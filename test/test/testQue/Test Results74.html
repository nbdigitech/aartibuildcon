  1. Consider a source and 32 destinations are connected in binary tree topology where each node is a router. Suppose that each link between 2 routers cost is C then what is cost to send a multicast packet in two approaches unicast emulation and network-layer-multicast. You can choose as many routers as you like.( Marks: -0.33 )
    1.   160
    2.   222
      Explanation:
      If sender wants to connect to 32 receivers through binary tree means, we will have 62 links with depth 5.
      With multicast, the packet traverses a link exactly once. So the cost of sending one packet to all 32 receivers via multicast is 62.

      With unicast, each of the 32 receivers is sent an individual packet. So each packet must travel across five links( depth of binary tree is 5) from the source to the destination. Hence the cost is 32*5=160.
      Then the total cost = 62 + 160 = 222
    3.   94
    4.   62


  2. Consider an IP link with maximum transfer unit of 700 bytes and each datagram size to send is 2400-bytes. Assume that the datagram which has been recently sent has identification number as 422. What is the total number of fragments has to be sent to transfer full datagram and what is the offset value of 2nd fragment from all the fragmented datagrams. Note : Consider the minimum header size of IPV4 datagram.( Marks: 1.00 )
    1.   4, 680
      Explanation:
      The Maximum transfer Unit = 700 bytes
      Each datagram size = 2400 bytes
      Minimum size of IPV4 header = 20 bytes

      2400 bytes = 2380 bytes data + 20 bytes header
      Then the datagram divides into multiple frames depend on MTU

      680 bytes data + 20 bytes header → fragment 1
      680 bytes data + 20 bytes header → fragment 2
      680 bytes data + 20 bytes header → fragment 3
      370 bytes data + 20 bytes header → fragment 4
      ⇒ Total number of fragments = 4
      Offset is the number of data bytes ahead of particular fragment.
      ⇒ Offset of 2nd fragment is 680 bytes
    2.   5, 370
    3.   4, 1320
    4.   1, 0

  7. Suppose a digital network implemented by circuit-switch technology which has a data rate of 1 Mbps. Assume that 2 machines are connected to this network and they are separated by 5000 Km with 2 routers in the path. If 1000 bits has to exchange between these 2 machines, What is the delay if the propagation speed of signal is 2 X 108 m/s? Assume that the setup phase is a two-way communication and the teardown phase is a one-way communication.( Marks: -0.66 )
    1.   101 ms
      Explanation:
      We assume that the setup phase is a two-way communication and the teardown
      phase is a one-way communication. These two phases are common for all three
      cases. The delay for these two phases can be calculated as three propagation delays
      and three transmission delays or
      3 [(5000 km)/ (2 ×108 m/s)]= 75 ms
      We assume that the data transfer is in one direction;
      the total delay is then
      delay for setup and teardown + propagation delay + transmission delay
      75 + 25 + 1 = 101 ms
    2.   102 ms
    3.   98 ms
    4.   92 ms


  8. Suppose a network consists of 2 host machines A and B. A’s each packets of length L has to travel upon 3 links which connected by 2 packet switches. Suppose that di, si, Ri are the length, propagation speed and the transmission rate of link i where i = link number(1, 2, 3). Suppose now the packet is 1,500 bytes, the propagation speed on all three links is 2.5 * 108 m/s, the transmission rates of all three links are 2 Mbps, the packet switch processing delay is 3 msec, the length of the first link is 5,000 km, the length of the second link is 4,000 km, and the length of the last link is 1,000 km. What is the end-to-end delay?( Marks: 2.00 )
    1.   68 ms
    2.   62 ms
    3.   64 ms
      Explanation:
      The first end system requires L/R1 to transmit the packet onto the first link
      The packet propagates over the first link in d1/s1.
      The packet switch adds a processing delay of dproc;
      After receiving the entire packet, the packet switch connecting the first and the second
      link requires L/R2 to transmit the packet onto the second link;
      The packet propagates over the second link in d2/s2. Similarly, we can find the delay caused by the second switch and the third link: L/R3, dproc, and d3/s3.
      Adding these five delays gives
      dend-end = L/R1 + d1/s1 + dproc + L/R2 + d2/s2 + dproc + L/R3 + d3/s3
      we simply plug the values into the equation to get
      6 + 20 + 3 + 6 + 16 + 3 + 6 + 4 = 64 msec.
    4.   72 ms

  10. Consider a client server scenario which uses TCP connection to communicate with each other. Assume the data has to send is 16 bytes, then what is efficiency of transmission at the TCP level, IP level and Data link layer level respectively. Note: no options are used in both IP header and Ethernet header.( Marks: 2.00 )
    1.   34, 31, 22
    2.   48, 31, 20
    3.   44, 29, 22
      Explanation:
      Efficiency = (Useful bytes / Total bytes)
      At TCP level:
      16 bytes of data / (16 bytes of data + 20 bytes of TCP header) ≈ 0.44 = 44 %
      At IP level
      16 bytes of data / (16 bytes of data + 20 bytes of TCP header + 20 bytes of IP header) ≈ 0.29 = 29 %
      At data link level (assuming no preamble or flag):
      16 bytes of data / (16 bytes of data + 20 bytes of TCP header + 20 bytes of IP header
      + 18 bytes of Ethernet header and trailer ≈ 0.22 = 22
    4.   22,29, 44

  13. Assume a router is forwarding IP datagrams each with a length of 1024 bytes which includes both header and payload. The datagrams will be lost permanently after 10 sec in the link. In this scenario what is the maximum data rate of link to avoid problem of having same ID numbers for 2 different datagrams?( Marks: 0.00 )
    1.   53677091 bps
    2.   53487091 bps
    3.   53617091 bps
    4.   53687091 bps
      Explanation:
      Let us assume b is the line speed. I.e b bits can transfer in 1 sec.
      Packet length = 1024 bytes or 1024 * 8 bits
      No. of packets can transfer = b/1024 * 8

      Within 10 sec, 216 packets can send (total no. of bits to represent ID = 16 and wrap around time = 10 sec).

      Transmission time for 1 packet Tt = (1024 * 8) / b sec
      To Transmit all the packet(216) = 216 * (1024 * 8)/b = 229/b sec

      Total transmission time for all packets is given as 10 sec because the Max. packet life time = 10 sec. So within 10 sec we need to send all the packets
      Thus 229/b = 10 sec
      ⇒ b = 53687091 bps
