  2. Which of the following method does not involve processor to transfer data between I/O module and main memory ?( Marks: 0.00 )
    1.   Programmed I/O
    2.   DMA
      Explanation:
      Programmed Input/Output (PIO) is a way of moving data between devices in a computer in which all data must pass through the processor. In programmed IO CPU issues a command then waits for I/O operations to be complete. As the CPU is faster than the I/O module, the problem with programmed I/O is that the CPU has to wait a long time for the I/O module of concern to be ready for either reception or transmission of data. The CPU, while waiting, must repeatedly check the status of the I/O module, and this process is known as Polling. As a result, the level of the performance of the entire system is severely degraded.
      In interrupt driven IO CPU issues commands to the I/O module then proceeds with its normal work until interrupted by I/O device on completion of its work.
      Direct Memory Access (DMA) means CPU grants I/O module authority to read from or write to memory without involvement. DMA module controls exchange of data between main memory and the I/O device. Because of DMA device can transfer data directly to and from memory, rather than using the CPU as an intermediary, and can thus relieve congestion on the bus.
    3.   Interrupt driven I/O
    4.   None of the above


  3. A disk unit has 24 recording surfaces and has a total of 14,000 cylinders. There is an average of 400 sectors per track. Each sector contains 512 bytes of data. What is the data transfer rate in bytes per second at a rotational speed of 7200 rpm?( Marks: 0.00 )

  4. Consider a disk drive with the following specifications: 32 surfaces, 256 tracks/ Surfaces, 128 sectors/ track and 256 Bytes/ sector. If the format overhead is 32 Bytes/ sector, what is the effective track capacity?( Marks: 0.00 )
    1.   32 KB
    2.   28 KB
      Explanation:
      Given disk drive has 32 surfaces
      256 tracks/surface
      128 sectors/track
      256 bytes/sector
      Format overhead=32 bytes/sector
      Track capacity=(Number of sectors/track)*(total bytes/sector)
      =128*256=215=32KB/track
      But given format overhead is 32 bytes/sector
      Total format overhead=128 sectors/track*32 bytes/sector
      =4KB/track
      Effective track capacity=32KB/track-4KB/track ⇒ 28 KB/track
    3.   38 KB
    4.   42KB

  7. What is the average time needed to read or write a 512-byte sector for a typical disk rotating at 15,000 RPM? The advertised average seek time is 4 ms, the transfer rate is 100MB/sec, and the controller overhead is 0.2 ms. Assume the disk is idle, so there is no waiting time.( Marks: 0.00 )
    1.   6.205 msec
      Explanation:
      Rotational speed=15,000 RPM
      Seek time=4 msec
      Data transfer rate=100MB/sec
      Controller overhead=0.2 msec
      Average time=seek time+rotational latency+data transfer time+controller overhead
      Rotational latency=½(rotational time)
      15,000 rotations----------->60 sec
      1 rotation------------->?
      ⇒ 1/250 sec=4 msec
      Rotational latency=½(4 msec)=2 msec
      Data transfer time=512 bytes/(100MB/sec)
      ⇒ 0.005 msec
      Average time=4 msec+2 msec+0.005 msec+0.2 msec=6.205 msec
    2.   4.205 msec

    4.   8.205 msec

  10. Consider a disk with a rotational rate of 10,000 RPM, an average seek time of 8 ms, and an average of 500 sectors per track. What is the average time to read a random sector from disk?( Marks: 0.00 )
    1.   9.012ms
    2.   17ms
    3.   11ms
    4.   11.012ms
      Explanation:
      Average time = seek time + rotational delay + transfer time
      Rotational delay=½(rotational time)
      10000 rotations----------->60 sec
      1 rotation--------------->?
      ⇒ 60/10000 sec ⇒ 6 msec


      The transfer time is the time it takes for the head to read all of the sector. The head is now at the start of the sector, so how long does it take for the entire sector to go past the head? Since there are on average 500 sectors per track, we need 1/500th of a revolution of the disk to read the entire sector. We can work this out as (time for one revolution of disk) / 500 = 6ms / 500 = 0.012ms.
      Average time=8 msec+½(6 msec)+0.012 msec=11.012 msec

  14. CommonData Question 14&15:
    Consider a system in which bus cycle takes 100 ns. Transfer of bus control from processor to device or device to processor, takes 250 ns. One of the IO device has data transfer rate of 75 KB/sec and employs DMA. Data are transferred one byte at a time.
    Suppose we employ DMA in a burst mode. That is, the DMA interface gains bus mastership prior to the start of block transfer and maintains control of the bus until the whole block is transferred. How long would the device tie up with the bus when transferring a block of 256 bytes?( Marks: 0.00 )
    1.   5.4 ms
    2.   6.4ms
    3.   2.4ms
    4.   3.4ms
      Explanation:
      Time to start + end DMA = 250 + 250= 500 nsec. Since device is slow, the DMA speed is governed by device speed.
      Time taken by device to transfer 256 bytes = (256/75 * 1024) sec = 3.4msec
      Total time taken = 500 nsec + 3.4 msec. ≈ 3.4 msec.


  15. CommonData Question 14&15:
    Consider a system in which bus cycle takes 100 ns. Transfer of bus control from processor to device or device to processor, takes 250 ns. One of the IO device has data transfer rate of 75 KB/sec and employs DMA. Data are transferred one byte at a time.
    How long would the device tie up with the bus when transferring a block of 256 bytes assuming DMA uses cycle stealing mode?( Marks: 0.00 )
    1.   0.153 ms
      Explanation:
      The bus cycle is stolen for 256 times.
      Time for bus transfers (256 times) = (250+250)*256 = 128,000 nsec.
      Time for 256 bus cycles = 256 * 100 = 25600 nsec.
      Total = 128000 + 25600 = 153600 ns = 0.153ms
    2.   0.546 ms
    3.   1.2ms
    4.   3.4ms
