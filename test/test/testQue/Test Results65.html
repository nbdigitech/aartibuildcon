  1. A 5-stage pipeline has the stage delays as 110, 110, 260, 160 and 140 ns respectively. Registers that are used between the stages have a delay of 5 ns each. Assuming constant clocking rate, the total time taken to process 200 instructions is _______________( Marks: 0.00 )
    1.   54060 ns
      Explanation:
      5-stage pipeline,
      Stage delays are 110, 110, 260, 160 and 140 ns.
      Register delay : 5 ns
      Number of instructions : 200
      Number of cycles required = k + (n – 1) clocks
      Where k = number of stages and 
       n = number of instructions
      Number of cycles required = 5 + (200 – 1) clocks
          = 5 + 199 clocks
          = 204 clocks
      Cycle Time = Maximum stage delay + Register delay
               = Max(110, 110, 260, 160, 140 ) + 5
      = 265ns
      Time required = 204 * 265 ns
          = 54,060 ns
    2.   23460 ns
    3.   44060 ns
    4.   34060 ns


  2. Consider a non-pipelined processor design which has a cycle time of 10ns and average CPI of 1.4. The maximum speedup pipelined processor can get by pipelining it into 5 stages and each stage takes 2ns is______________?( Marks: 0.00 )
    1.   5
    2.   6
    3.   7
      Explanation:
      let the number of instructions be (n)
      let t1 be time taken without pipeline to execute (n=number of instructions) instruction

      cpi=1.4 number of cycles to execute one instruction

      t1=cpi*n*cycle time=1.4 * n*10 which is time tajenen to execute (n) instructions in non-pipeline system
      let t2 be time taken with pipeline to execute (n=number of instructions) instruction
      cycle time for pipeline system=2
      t2=2*5+(n-1)*2=2*n+8 time taken to execute n instruction in pipelined system.

      speedup= t1/t2=(1.4*n*10)/(2*n+8)
      when n is very large the speed up is equal to = (1.4*n*10)/(2*n)=(1.4*10)/2=7
    4.   8

  6. Consider a pipelined processor having 5 stages. The stage delays are 1, 3.2, 2, 1.5 and 1 ns. The 3rd stage is capable of deciding branch target address. Processor starts fetching new instruction when the conditional branch outcome is known. 40% of instructions are conditional branch, then calculate the execution time for 103instructions?( Marks: 0.00 )
    1.   4670ns
    2.   4870ns
    3.   5670ns
      Explanation:
      5 stage pipeline
      Delays are 1, 3.2, 2, 1.5 and 1 ns
      40% instructions are Branch

      Clock cycle time = max (stage delay) = 3.2 ns
      Number of clock cycles = (1 + stall Freq + stall cycles) clocks
      = (1 + 0.4 * 2) clocks
      = 1.8 clocks
      Execution time = 1.8 * 3.2 ns / instruction
      = 5.76 ns / instruction
      Execution time for 103 instructions = 5.76 ns / instruction * 103 instruction
      = 5760 ns
    4.   1430ns


  7. Consider a non-pipelined processor design which has a cycle time of 10ns and average CPI of 1.4. If each pipeline stage adds extra 20ps due to register setup delay. The pipeline stalls 20% of the time for 1 cycle and 5% of the time for 2 cycles (these occurrences are disjoint). What is the new CPI?( Marks: 0.00 )
    1.   1.7
      Explanation:
      Cycle time is 10 ns
      Average CPI = 1.4
      Register delay = 20 ps
      20% of time for 1 cycle = 1.4 + 1 = 2.4
      5% of time for 2 cycles = 1.4 + 2 = 3.4
      75% of time = 1.4
      New CPI = 2.4 * 0.2 + 3.4 * 0.05 + 1.4 * 0.75
      = 0.48 + 0.17 + 1.05
      New CPI = 1.7
    2.   1.8
    3.   1.9
    4.   2.7


  8. Consider a pipelined processor having 5 stages and operated at 2GHZ clock frequency. The 5 stages are IF, ID, IE, MEM and WB respectively and each stage takes equal amount of time to complete. The processor received 20% of instructions are branch, then calculate the average instruction time of the pipelined processor ?( Marks: 0.00 )
    1.   1.8ns
    2.   2.6ns
    3.   0.9 ns
      Explanation:
      5 stage pipeline
      Clock frequency = 2 GHz
      20% of instructions are Branch
      Number of stall cycle = k – 1 = 5 – 1 = 4
      Number of clock cycles = (1 + stall Freq * stall cycles)
      = (1 + 20% * 4) clocks
      = (1 + 0.2 * 4) clocks
      = (1 + 0.8) clocks
      = 1.8 * 2 GHz / instruction
      = 1.8 * 0.5 ns / instruction
      = 0.9 ns / instruction
      0.9 ns
    4.   4.8ns

  11. Suppose that an n-segment pipeline executes m instructions, and that a fraction fstall of the instructions require the insertion of k stalls per instruction to resolve data dependencies. Calculate the number of clocks required for m instructions ?( Marks: 0.00 )
    1.   m(1+k)clocks
    2.   m(1+fk)clocks
      Explanation:
      n-segment pipeline
      Number of instructions = m
      Number of stall cycles = k
      Stall frequency = f
      CPI (clocks per instruction) = (1 + stall Freq * stall cycles)
      = (1 + f * k) cycles / instruction
      For ‘m’ instructions = (1 + f * k) * m cycles
      = m + mfk
      = m(1 + fk) clocks
    3.   m+mf clocks
    4.   m(f+k)clocks

  13. A pipeline system overlap all kind of instructions except branch instructions. Branch instructions introduces 3 stall cycles. 60% of total instructions are branch, what will be the throughput of the pipeline system assuming the clock cycle time is 5ns?( Marks: 0.00 )
    1.   50 MIPS
    2.   70 MIPS
      Explanation:
      Number of stall cycles = 3
      Stall frequency = 60%
      Clock cycles time = 5 ns
      CPI = (1 + stall Freq * stall cycles) clocks / instruction
      = (1 + 0.6 * 3) clocks / instruction
      = (1 + 1.8) clocks / instruction
      = 2.8 clocks / instruction
      Average time taken to execute instruction = 2.8 * 5 ns
      = 14 ns
      14 ns → 1 instruction
      1 sec → ?
    3.   40 MIPS
    4.   60 MIPS
