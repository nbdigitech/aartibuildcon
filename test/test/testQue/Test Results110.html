  5. Consider a paging system with page size of 8 KB. If a process of size 34 KB is in logical address space, find the internal fragmentation in Kbytes.( Marks: 0.00 )
    1.   4
    2.   5
    3.   6
      Explanation:
      Size of page = 8KB
      Size of process = 34 KB
      Number of pages required by process = 34KB/8KB=4.25
      So, the number of pages required by the process is 5
      Now, the address space allocated for the process is 5 * 8 = 40 KB
      But required = 34 KB
      ∴ 40 – 34 = 6 KB is the internal fragmentation.
    4.   7

  14. Consider a system using multi level paging environment. Address bits are segregated as follows: Here first 8 bits represent the required bits for first level, next 8 bits represent the required bits for second level, next 6 bits represent the required bits for third level and last 10 bits represents the offset.

    What is the size of memory required for storing the all page tables for a process that has 512 K of memory starting at address 0? Assume each page table entry is 4 bytes.( Marks: 0.00 )
    1.   2048
    2.   3192
    3.   1024
    4.   4096
      Explanation:
      Given

      Size of page = 210 = 1024 bytes = 1KB
      Number of pages = = 222
      Given that size of page table entry = 4 bytes
      According to given division
      1st level page table contains: 256 entries
      2nd level page table contains: 256 entries
      3rd level page table contains: 64 entries
      Given process needs: 512K of memory
      Number of pages needed by a process = = 512
      ∴ We need eight 3rd level page tables
      ∴ We need 8 entries in 2nd level page table
      ∴ We need 1 entry in 1st level page table
      Size: (8 * 64 * 4) + (256 * 4) + (256 * 4)
      = 211 + 210 + 210
      = 2048 + 2048
      = 4096 Bytes
