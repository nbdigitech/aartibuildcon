  44. A relational schema R (A, B, C, D, E) holds following FDs. A → BC, CD → E, B → D, E → A. R is decomposed into R1 (A, B, C) and R2 (A,D,E) then the decomposition is( Marks: 0.00 )
    1.   Lossless and dependency preserving
    2.   Lossy and dependency preserving
    3.   Lossless and not dependency preserving
      Explanation:
      FD’s under sub relation R1 (A, B, C) are A → BC
      FD’s under sub relation R2 (A, D, E) are A → DE, E → AD.
      ∴ Key for R1 is A
      Key for R2 is A, E
      Common attribute between R1 and R2 is A which is key for R1 and R2. So, decomposition is lossless.
      Checking whether decomposition is FD preserving or not:
      Initial FD set: F: {A → BC, CD → E, B → D, E → A}
      Final FD set G: {A → BC, A → DE, E → AD}
      A → BC in F is covered by G
      CD → E in F is not covered by G since (CD)+ in G is CD only.
      So, decomposition is not FD preserving
      Answer is option (c).
    4.   Lossy and not dependency preserving

    2.   Only S2 is true.
    3.   Both S1 and S2 is true.
      Explanation:
      Justification for S1:
      If any language L is recursively enumerable, then there is a Turing machine that semidecides it. Every Turing machine has a description of finite length. Therefore, the number of Turing machines, and thus the
      number of recursively enumerable languages, is countably infinite (since the power set of a countable set is countably infinite). If, for some language L, its complement is recursively enumerable, then it must have a semideciding Turing machine, so there is a countably infinite number of languages whose complement is recursively enumerable. But there is an uncountable number of languages. So there must be languages that are not recursively enumerable and do not have recursively enumerable complements.
