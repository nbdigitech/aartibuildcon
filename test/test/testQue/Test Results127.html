      And every identifier entry is made in symbol table along with its scope.

  9. Consider the following three statements:
    S1: The lexical analyzer detects an error when it discovers that an input prefix does not fit the specification of any token class.
    S2: The parser can help the lexical analyzer improve its ability to recover from errors by making the list of legitimate tokens available to the error recovery routine.
    S3: The lexical analyzer can detect type checking errors.( Marks: 0.00 )
    1.   Only S1 is true
    2.   Only S3 is true
    3.   S1, S2, S3 are true
    4.   S1 and S2 are true
      Explanation:
      Justifying statement 1:
      The tokens in C language include:
      (i) Keyword
      (ii) Identifier
      (iii) Constant
      (iv) String-literal
      (v) Punctuator
      (vi) Operators
      If there is an invalid token which does not belong to the 6 types of tokens shown above we get a lex error
      Justifying statement 2:
      Once the parser makes the list of legitimate tokens available to the error recovery routine then this routine can decide whether a remaining input’s prefix matches one of these tokens closely enough to be treated as that token. This helps the lexical analyzer.
      Statement 3:
      This is false as type checking is done in semantic analysis phase
