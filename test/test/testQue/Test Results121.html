  3. In an E-R diagram, the entry set person has UID, age, sex and address as attributes wit UID as primary key. Address is a multi valued composite attribute comprised of street number, city and pincode. When this entity set is mapped to a relational model, the relations are( Marks: 1.00 )
    1.   Person(UID, age, sex, address[n])
    2.   Person(UID, age,sex), Person_address(UID, street number, city, pincode)
      Explanation:
      We know that for each multi valued attribute A, we create a new relation R. This relation R will include an attribute corresponding to A, plus the primary key attribute K of the relation that represents the entity type or relationship type that has A as an attribute. The primary key of R is the combination of A and K. If the multivalued attribute is composite, we include its simple components. In our example we create a new relation Person_address. The set of attributes in relation Person_address {street number, city, pincode } represents the multivalued composite attribute address of Person, while UID as foreign key in Person_address referring to UID in Person. The primary key of Person_address relation is {UID, street number, city, pincode}.
    3.   Person(UID, age, street number[n], city[n], pincode[n])
    4.   None of the above

  6. Consider an empty relation R(AB) where A is primary key and B is foreign key referring same relation R.
    Which one of the following row sequence can be inserted into R?( Marks: 2.00 )
    1.   (1, 2) (2, 3) (3, 4) (4, 5)
    2.   (1, NULL) (2, 1) (3, 2) (4, 3)
      Explanation:
      Option (a) is wrong because, when we try to insert the tuple/row (1, 2) into the relation (A, B), it will through an error because attribute B is referring to attribute A and attribute A does not have the value 2. It will also throw an error when we try to insert the tuple/row (4,5) into the relation (A, B), because A does not has value 5.

      Option (b) is correct as it does not throw any error while inserting tuples/rows into relation as it follows referential integrity constraint. For referential integrity to hold in relational database, any field in a table that is declared a foreign key can contain either a null value or only values from a parent table’s primary key or a candidate key.

      Option (c) is wrong because, when we try to insert the tuple/row (4, 3) into the relation (A, B) it will through an error because attribute B having the value 3 is referring to attribute A and attribute A does not have the value 3.
    3.   (1, NULL) (2, NULL) (4, 3) (5, 4)
    4.   None of the above

  12. Which of the following statement(s) is/are correct?
    I.A constraint showing the maximum number of entities to which another entity can be associated via a relationship set is called participation constraint.
    II.The number of entity sets participating in the relationship is known as the degree of that relationship.( Marks: -0.66 )
    1.   Only I is correct
    2.   Only II is correct
      Explanation:
      1.A constraint showing the maximum number of entities to which another entity can be associated via a relationship set is called cardinality constraint.
      For example one entity can be associated with at most n entities via relationship set then cardinality ratio of this relationship will be 1:N.

      2.The number of entity sets participating in the relationship is known as the degree of that relationship.
      For example if two entity sets are participating in a relationship, then degree of that relationship is “2”.
    3.   Both I & II are correct
    4.   None of the above.


  13. Which of the following statement(s) is/are true?

    III.The relationship between owner entity and the weak entity is one to many.( Marks: -0.66 )
    1.   Only II & III are correct
      Explanation:
      Statement I:
      Every existence dependency need not result in a weak entity.
      For example: A subject being offered at the university, must have the relevant offering details associated with it, so OFFERING is existence-dependent on SUBJECT. But here OFFERING has its own primary key (offering#) which has not been derived from SUBJECT. Therefore OFFERING is not a weak entity. So, statement I is not correct.

      Statement II:
      Roles names are mandatory in relationship types where all the participating entity types are same. In this case we concatenate role indicator with attribute from the entity set to get role name. So, statement II is correct.

      Statement III:
      The relationship between owner entity set and weak entity set is one to many

The cardinality ratio between employee and dependent is 1: N. this means each employee has n dependents but every dependent is dependent of one employee only. So, statement III is correct.
