  1. Which of the following properties are required to solve dynamic programing( Marks: 1.00 )
    1.   Overlapping subproblems
    2.   Optimal Substructure
    3.   Both (a) and (b)
      Explanation:
      Dynamic programing is an algorithm paradigm that solves a given complex problem by breaking it into sub problems and stores the results of subproblems to avoid computing the same results again. To solve dynamic programing problems we need following two properties

      (1) Overlapping Subproblems
      (2) Optimal Substructure

      Consider the following example
      int fib(int n)
      {
      if(n<=1)
      return n;
      return fib(n-1) + fib(n-2);
      }
      fib(4) = fib(3) + fib(2)
      fib(3) = fib(2) + fib(1)
      fib(2) = fib(1) + fib(0)
      Here we are computing same sub problem again and again i.e overlapping sub problem.
    4.   None of the above

  3. Which of the following statement is/are TRUE?

    1.   Only I
    2.   Only II
      Explanation:
      I. False: The larger problems depend on the smaller ones, so the smaller ones need to be solved first. The smaller problems get solved (and memoized) recursively as part of a larger problem.

      II. True: The time complexity of Bellman-ford algorithm is O(VE)

      Algorithm:
      Step 1: Initialize distances from source to all vertices as infinite and distance to source itself as 0. Create an array Temp[] of size |V| with all values as infinite except Temp[source] where source is source vertex.
      Step 2: Calculate shortest distances. Do following |V|-1 times where |V| is the number of vertices in the given graph.

      (a) Do following for each edge u – v
      if Temp[v] > Temp[u] + weight of the edge uv, then update dist[v]
      Temp[v] = Temp[u] + weight of edge uv.
      Step 3: if there is a negative weight cycle in graph. Do following for each edge u-v
      if Temp[v] > Temp[u] + weight of edge uv, then “Graph contains negative weight cycle”
    3.   Both I and II
    4.   Neither I nor II


  4. What is the time complexity of traveling salesperson problem with n vertices using dynamic programming?( Marks: -0.33 )
    1.   O(n2n)
    2.   O(n log n)
    3.   O(n2 2n)
      Explanation:
      Let us define a term Cost(S, i) be the cost of the minimum cost path visiting each vertex in set S exactly once, starting at 1 and ending at i.

      We start with all subsets of size 2 and calculate Cost(S, I) for all subsets where S is the subset, then we calculate Cost(S, I) for all subsets S of size 3 and so on.
      If size of S is 2, then S must be {1, i},
      Cost(S, i) = Distance(1, i)
      Else if size of S > 2
      Cost(S, I) = min { Cost(S-{i}, j) + distance(j, I)} where j belongs to S, j != i and j != 1

      There are at most O(n*2n) subproblems, and each one takes linear time to solve. Total time is O(n2 *2n)
    4.   O(n2 log n)
