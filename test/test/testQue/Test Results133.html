  1. Consider the following statements
    S1] Multi tape - TM is more powerful than Single tape - TM
    S2] If algorithm exists for a particular problem then the language for the problem is Recursive
    Which of the following is TRUE?( Marks: 1.00 )
    1.   S1 is TRUE & S2 is TRUE
    2.   S1 is TRUE & S2 is FALSE
    3.   S1 is FALSE & S2 is FALSE
    4.   S1 is FALSE & S2 is TRUE
      Explanation:
      S1: Time taken for parsing a string using multi tape - TM and that of single tape - TM may vary but not the accepting power.
      Number of languages accepted by multi tape - TM is equal to the number of languages accepted by single tape - TM

      S2:The concept of an algorithm requires the problem to be solved in finite time producing a definite result following a predefined procedure. The set of recursive languages are those which halt on all inputs, thus either comes up with YES or NO. In recursive enumerable sets, strings accepted procedure a YES result but on some strings the corresponding TM may go on forever thus violating the requirements of an algorithm.

  4. L = {<m,w> | m accepts w}, m is a TM and w is a string.( Marks: 1.00 )
    1.   L is recursive
    2.   L is recursive enumerable but not recursive
      Explanation:
      Given a string w and a Turing machine M, there exists no algorithm which decides whether the string w is accepted or not. So, it is not recursive.

      The Turing machine M may or may not halt over the string w, so it is recursively enumerable. (The membership property for TM is undecidable).

      Now, let’s see what type of strings the grammar can generate.
      S derives ε from S → T, T → ε
      S derives bc:

  10. Consider the Turing machine below:

    Which is the following language accepted by given TM( Marks: 2.00 )
    1.   {anbm | n < m}
    2.   {anbm | n > m }
    3.   {anbm | n = m }
      Explanation:
      1. The given Turing machine on seeing a, converts it to x, goes to q1 and moves the read/write header to right.
      2. At q1, it skips any number of a’s and moves right, being at q1 only.
      3. On seeing b at q1, it moves left by making it y and transits to q2.
      4. At q2, it skips all a’s and moves left being at q2 until it reaches the output symbol x on the tape.
      5. Once it reaches the output symbol x, it skips and moves right and transits to q0.
      Till now, we are done with one round of parsing the whole string.
      In the next rounds, the above procedure repeats till it converts every a into x and every b into y. If we find any output symbol y, we simply skip it and move right being at q1. Once we convert all a’s into x, then we will skip all y’s and move to the right being at q3.
      Observe that, we are actually counting the number of a’s and b’s. If all a’s are followed by b’s and the number of a’s and b’s are equal, then only we are accepting the string. Otherwise, we do not accept the strings. See that strings like aaabb, aabbb, abab are not accepted.

      Consider the string ab
      q0abB → xq1bB → xq2yB → q2xyB → xq0yB → xyq3B → xyq3B → xyBqf
      ab is accepted
    4.   {w|na(w)=nb(w)}

  13. Match the following languages
    ( Marks: 2.00 )
    1.   1 – (b), 2 – (b), 3 – (c)
    2.   1 – (c), 2 – (c), 3 – (a)
    3.   1 – (a), 2 – (a), 3 – (c)
    4.   1 – (b), 2 – (a), 3 – (c)
      Explanation:
      L={w1m|w∈{a,b}* and na(w)=m}
      Push all a’s and b’s on to the stack. Pop from the stack on seeing 1. If the stack becomes empty and the string is completely parsed then the string is in the language. Otherwise it is not in the language. We need only one stack and we can define every transition deterministically. So, this is DCFL.

      L= {0n1n0n|n≥0}
      We need two stacks to recognize the string whether it is in the language or not. It is CSL.

      L={anb2n}∪{anbn}
      We can construct DPDAs for the languages {anb2n } and {anbn}. But the union of these two languages cannot be deterministic. We need to guess whether the string is of the form {anbn}or of the form {anb2n}. This is non deterministic context free language.
