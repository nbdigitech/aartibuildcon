  13. Which of the following instruction is following indirect mode addressing mode?( Marks: 0.00 )
    1.   LD 1000
    2.   Move R1, R2
    3.   LD (R1)
    4.   LD @ADR
      Explanation:
      Addressing modes refers to the different ways an instruction can specify an operand.
      Immediate Addressing Mode:
      The operand (data) immediately follows the instruction (the data is in the byte(s) that immediately follow the instruction) This type of addressing is mainly used when the operand is a constant and its value is known at the time the program is written.
      Assembly convention: LD 1000
      Register transfer: AC<-1000
      Indirect addressing mode:
      In this mode, the address field of an instruction gives not an actual operand address instead it gives the address where the effective address is stored in memory
      EA[Effective address] = address contained in memory location M
      Assembly convention: LD @ADR
      Register transfer: AC<- M[M[ADR]]
      Register Indirect addressing mode:
      In this mode the instruction specifies a register in the cpu whose content give the address of the operand in memory. In other words, the selected register contains the address of the operand rather than the operand itself.
      Assembly convention: LD (R1)
      Register transfer: AC<-M[R1]
      Register addressing mode:
      Operand is held in register named in the address field.
      EA[Effective address] = R
      MOV R1 R2 is in register addressing mode format

  17. A 5-stage pipeline has the stage delays as 110, 120, 280, 160 and 150 ns respectively. Registers that are used between the stages have a delay of 5 ns each. Assuming constant clocking rate, the total time taken to process 200 instructions is _______________( Marks: 0.00 )
    1.   58,140
      Explanation:
      5-stage pipeline,
      Stage delays are 110, 120, 280, 160 and 150 ns.
      Register delay : 5 ns
      Number of instructions : 200
      Number of cycles required = k + (n – 1) clocks
      Where k = number of stages and
      n = number of instructions
      Number of cycles required = 5 + (200 – 1) clocks
      = 5 + 199 clocks
      = 204 clocks
      Cycle Time = Maximum stage delay + Register delay
      = Max(110, 120, 280, 160, 150 ) + 5
      = 285ns
      Time required = 204 * 285 ns = 58,140
    2.   54,060
    3.   20,440
    4.   64,100

  19. Consider a disk pack with the following specifications 16 surfaces,256 tracks/surface,128 sectors/track and 256 bytes/sector. in the above disk pack formatted overhead 64 bytes/sector then what is the formatted disk space?( Marks: 0.00 )
    1.   38MB
    2.   48MB
    3.   66MB
    4.   96MB
      Explanation:
      Disk pack consist of 16 surfaces
      256 tracks/surface,128 sectors/track and 256 bytes/sector.
      Given formatted overhead=64 bytes/sector
      Total capacity of the disk pack=16 surfaces*256 tracks/surface*128 sectors/track*256 bytes/sector ⇒ 128 MB
      Formatted overhead of the sectors=number of sectors * formatted overhead
      = 16 surfaces * 256 tracks/surface * 128 sectors/track * 64 bytes/sector
      =32 MB
      Formatted disk space=128 MB - 32 MB=96 MB

  22. How many memory accesses required by the following instructions?
    SUB r1, r2, r3
    MUL r1, r2, (r3)
    ADD r1, r2, (r3)
    DIV r1, r2, @(r4)

    Suppose every instruction is one word long, as well as every address.( Marks: 0.00 )
    1.   4
    2.   6
    3.   9
    4.   8
      Explanation:
      SUB r1, r2, r3 require only one memory access, reading the instruction. MUL r1, r2, (r3) require two memory accesses, the first to read the instruction and the other one to read the value from memory location(s), whose address is in r3, similarly two memory accesses for ADD r1, r2, (r3). DIV r1, r2, @(r4) three memory accesses are made in this case, the first to read the instruction, the second to get M[r4], and the third one to get M[M[r4]]. So total memory accesses required are = 1 + 2 + 2 + 3 = 8.
