  1. Choose the wrong statements below.
    A.Host A is sending a large file to host B over a TCP connection. Assume host B has no data to send to host A. Host B will not send acknowledgments to host A because host B cannot piggyback the acknowledgments on data.
    B.The size of the TCP RcvWindow never changes throughout the duration of the connection.
    C.Suppose host A is sending a large file to host B over a TCP connection. The number of unacknowledged bytes that A sends cannot exceed the size of the advertised receiver buffer
    D.Suppose that the last SampleRTT in a TCP connection is equal to 1sec. Then the current value of TimeoutInterval for the connection will necessarily be >= 1 sec.( Marks: -0.33 )
    1.   A only
    2.   A and B only
    3.   A, B and D only
      Explanation:
      A.Piggybacking is an optimization that is used when both sides have to send data to each other so that the receiver, instead of sending two packets i.e., an ACK and a data packet, it just sends one. When the receiver (B) does not have any data to send, it will still send an ACK with the sequence number field containing the next sequence of data it is supposed to send.
      B.RcvWindow is used to give the sender an idea of how much free buffer space is available at the receiver. Thus as the amount of unacknowledged TCP data varies the RcvWindow also changes.
      C.TCP is not permitted to overflow the allocated receiver buffer. Hence when the sender can not send any more data RcvWindow would be 0 and hence all the buffer would have unacknowledged data.
      D.Depending on the value of EstimatedRTT it may or may not be greater than 1.
    4.   All

To find DevRTT, assuming a value of β = 0.25 and assuming the value of DevRTT was 5 ms just before the first of these three samples was obtained.     2.   120 ms

  10. Suppose Host H1 and H2 are connected over TCP connection with a 100Mbps link. Host H1 is sending a random file over this connection. H1 can send the data into sockets at rate of 120 Mbps but H2 can only receive at maximum rate of 50 Mbps due to limit on buffers at H2. What is the sender effective bandwidth utilization?( Marks: 2.00 )
    1.   60 Mbps
    2.   40 Mbps
    3.   50 Mbps
      Explanation:
      Since the link capacity is only 100 Mbps.
      so host H1’s sending rate can be at most 100 Mbps. Still, host H1 sends data into the receive buffer faster than Host H2 can remove data from the buffer.
      The receive buffer fills up at a rate of 50 Mbps.
      When the buffer is full, Host H2 signals to Host H1 to stop sending data by setting RcvWindow = 0.
      Host H1 then stops sending until it receives a TCP segment with RcvWindow > 0.
      Host H1 will thus repeatedly stop and start sending as a function of the RcvWindow values it receives from Host H2.
      On average, the long-term rate at which Host H1 sends data to Host H2 as part of this connection is no more than 50 Mbps
    4.   75 Mbps


  11. Suppose a packet of length L bytes transfer from Host H1 to Host H2 over a 155 Mbps link. The header size in data-link and network and transport layers is 22 bytes each and MSS is 536 bytes. Consider transferring an enormous file of L bytes from Host A to Host B. Find how long it takes to transmit the full packet. Assume that H1 is sending segments continuously.( Marks: 0.00 )
    1.   148
    2.   228
    3.   155
    4.   249
      Explanation:
      There are 232 possible sequence numbers.
      The sequence number does not increment by one with each segment. Rather, it
      increments by the number of bytes of data sent. So the size of the MSS is irrelevant --
      the maximum size file that can be sent from A to B is simply the number of bytes
      representable by 232 = 4.19 Gbytes
      The number of segments is = 232 / 536 = 8,012,999.
      66 bytes of header get added to each segment giving a total of 528,857,934 bytes of header. The total number of bytes transmitted is 232 + 528,857,934 = 4.824*109 bytes.
      Thus it would take 249 seconds to transmit the file over a 155 Mbps link.

  14. Suppose a TCP connection uses a fifteen bit clock to generate starting sequence numbers. The clock ticks for each 0.1 sec and each tick generate a sequence number. The lifetime of a packet 60 sec. If 300 sequence numbers are used by packets for each minute, what is the time for resynchronization of clock?( Marks: 0.00 )
    1.   3261.6 sec
    2.   4261.6 sec
    3.   5361.6 sec
    4.   6433.6 sec
      Explanation:
      Total number of ticks before the clock cycles around is 215 = 32768 ticks
      Clock ticking time = 0.1 sec.
      The time elapsed for the clock to cycle around = 32768 × 0.1 = 3276.8 sec
      Max packet lifetime T = 60 sec
      Assume at "t" clock will resync
      At 300 sequence numbers/min means 5 sequence numbers/sec. So, the actual sequence number is 5*t [where t is in sec].
      Clock ticks per sec = 1/0.1
      The sender of the forbidden region(keep idle) is (1/0.1)(t − (3276.8-60)).
      Equating these two formulas, 5t = (1/0.1)(t − 3216.8)
      Thus, we find that they intersect at t = 6433.6 sec.
