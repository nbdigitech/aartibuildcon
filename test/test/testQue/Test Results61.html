  5. Consider the program which was made to run on two different machines the machine 2 results 25% speeder however the CPI is increased by 20%.If the clock frequency of the machine 1 is 1 MHZ what is the clock frequency of machine 2?( Marks: 0.00 )
    1.   1.6 MHZ
      Explanation:
      Let X is the time for program on machine 1
      And Y is the time for program on machine 2
      Given that Y=X-25%(X)
      Y ⇒ ¾(X)
      X=number of instructions * CPI1 * Tclock1
      Y=number of instructions * CPI2 * Tclock2


      Given that CPI2=CPI1 + 20%(CPI1)
      CPI2 ⇒ 1.2(CPI1)
      X/Y=4/3
      (number of instructions * CPI1 * Tclock1) / ( number of instructions * CPI2 * Tclock2)=4/3

      ⇒ (CPI1*(1/frequency 1)) / (1.2 CPI1 * (1/frequency 2)) = 4/3


      ⇒ (frequency 2) / (frequency 1) = (4/3) *(1.2) ⇒ 1.6


      ⇒ frequency 2=1.6 * frequency 1
      ⇒ frequency 2 = 1.6 * 1MHZ
      ⇒ frequency 2 = 1.6 MHZ
    2.   2.6 MHZ
    3.   3.6 MHZ
    4.   4.6 MHZ

  23. Computer A has an overall CPI of 1.3 and can be run at a clock rate of 600MHz. Computer B has a CPI of 2.5 and can be run at a clock rate of 750 Mhz. We have a particular program we wish to run. When compiled for computer A, this program has exactly 100,000 instructions.     1.   64000
    2.   65000
      Explanation:
      clock cycle time = 1/(frequency)
      (CPUTime)A = (Instruction count)A * (CPI)A * (Clock cycle Time)A = (100,000)*(1.3)/(600*106 ) ns
      (CPUTime)B = (Instruction count)B * (CPI)B * (Clock cycle Time)B = (I)B*(2.5)/(750*106) ns
      Since (CPUTime)A = (CPUTime)B,
      we have to solve for (I)B and get 65000

  25. Consider a device that is to transfer data at a rate of 1 Mbyte/sec in 16 byte chunks. The overhead of polling is 400 cycles on 500 Mhz processor. Calculate the overhead of polling assuming I/O uses polling mechanism to transfer data?( Marks: 0.00 )
    1.   5%
      Explanation:
      The polling rate is now (1 * 220 /16) since service is needed only for every 16 byte chunks. The poll consumes 400 cycles, to transfer (1 * 220 /16) chunks polling consumes 400 *216 cycles.
      Since the processor is 500 Mhz, the total number of available cycles is 500 * 106cycles.
      Overhead (% of cpu cycles consumed) = (cycles consumed for polling) * 100/ (Total CPU cycles)
      = (400 *216) * 100/ (500 * 106)
      = 0.05 * 100
      = 5%
