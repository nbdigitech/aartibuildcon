  19. Consider an array containing both +ve and -ve numbers. What is the tightest upper bound to find the two elements such that their sum is closet to ‘0’?
    Example: 50 -20 60 -90 95 gives -90 and 95( Marks: 0.00 )
    1.   O(nlogn)
      Explanation:
      Algorithm:
      Step1: Sort the array (takes nlogn time)
      Step 2: Set positive_closest = INT_MAX;
      Set negative_closest = INT_MIN;

      Step 3:
      i = 0, j = n -1
      while(i<j)
      {
      temp = A[i] + A[j];
      if(temp > 0)
      {
      if(temp < positive_closest)
      {
      positive_closest = temp;
      j --;
      }
      }
      else if(temp < 0)
      {
      if(temp > negative_closest)
      {
      negative_closest = temp;
      i++;
      }
      }
      }

      Time complexity is = O(nlogn) + O(n) = O(nlogn)
    2.   O(n2)
    3.   O(n3)
    4.   O(n)
