    1.   Insertion Sort with time complexity O(Kn)
    2.   Heap Sort with time complexity O(nLogk)
      Explanation:
      Efficient Method:
      1. Create a Min Heap of size k + 1 with first k + 1 elements. This will take O (k) time.
      2. One by one we have to remove min element from heap, put it in result array and add a new element to heap from remaining elements.
      In second step we need to do total ‘k’ extract min and (n-k) replace Min.
      Extract min means extracting minimum and min- heapify the remaining heap → O(log k) (since size of heap is k)
      Replace min means we have to first to extract min (i.e.,root element) and replace root with new element and do min-heapify of the resultant heap → O(log k) time
      So, total time for step 2 is k log k + (n-k) log k = n log k
      Therefore total time complexity = O(n log k)
