  7. What is the runtime complexity to detect deadlock in banker's algorithm with n processes and m resources?( Marks: 0.00 )
    1.   O(mn2)
      Explanation:
      Let work be an integer array of length m, initialized to available.
      Let finish be a boolean array of length n.
      For all i, if allocation[i] != 0, then finish[i] = false;
      Otherwise finish[i] = true.
      Find an i such that both
      finish[i] == false // Pi is currently not involved in a deadlock
      request[i] <= work
      If no such i exists, go to step 4
      // reclaim the resources of process Pi
      work = work + allocation[i];
      finish[i] = true;
      Go to step 2
      If finish[i] == false for some i,
      Then the system is in a deadlocked state.
      Moreover, if finish[i] == false, then process Pi is deadlocked.
      Run-time complexity: O(mn²).
    2.   O(m2n)
    3.   O(n2)
    4.   O(m2)


  8. Suppose there are ‘K’ processes, all the processes share total of ‘L’ identical resource units, which can be reserved and released one at a time. Each process requires a maximum of ‘M’ resource units, where M > 0. Which one of the following is a necessary condition for ensuring that deadlock does not occur?( Marks: 0.00 )
    1.   M(K-1)+1<L
    2.   K(M-1)+1>L
    3.   KM-1<L
    4.   K(M-1)+1<=L
      Explanation:
      From the given information we have K processes and each process required M resources.
      ∴ The maximum number of resources allocated that will cause deadlock is
      K(M – 1)
      ∴ By adding 1 more resource deadlock will be avoided.
      ∴ Minimum number of resources required for deadlock free is K(M – 1) + 1
      From the given data total number of resources available is 1. Hence the necessary condition is
      K(M – 1) + 1 <= L.
