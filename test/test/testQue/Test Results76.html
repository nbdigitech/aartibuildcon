  1. The minimum size of ethernet frame is 64 bytes long to make sure that the sender is able to detect the collision at the far end of the cable. If the Fast Ethernet with bandwidth 100 Mb/s also has the same 64 byte minimum frame size, but can get the bits out ten times faster. How is it possible to maintain the same frame size?( Marks: 1.00 )
    1.   increase the velocity of cable by a factor of 10
    2.   reduce the cable length by a factor of 10
      Explanation:
      In order to detect collision on the channel, the transmission time Tt > 2*Tp. So for the faster Ethernet, since the transmission rate is increased by a factor of 10, we will have to decrease the propagation delay by 10 so that it can still detect the collision. So the maximum wire length in Fast Ethernet is 1/10 as long as in ethernet.
      ⇒ Tt ∝ Tp
      ⇒ Length/Bandwidth ∝ Distance/Velocity
      ⇒ Bandwidth ∝ 1 / Distance(length)
    3.   increase the cable length by a factor of 10
    4.   Frame size must be change


  2. Consider a scenario that a communication link having a bandwidth of 10 Mbps with 20 routers and each router having a 3μs queuing delay and 2 μs processing delay. The length of cable is 3000 Km and speed inside the link is equal to speed of light. Then what is total delay to send a packet of size 25 million bits? Ignore the transmission delay at routers.( Marks: 0.00 )
    1.   3.7 sec
    2.   3.5 sec
    3.   2.6 sec
      Explanation:
      Latency = processing time + queuing time + transmission time + propagation time
      Processing time = 20 × 2 μs = 40 μs = 0.000040 s
      Queuing time = 20 × 3 μs = 60 μs = 0.000060 s
      Transmission time = 25,000,000 bits / (10 Mbps) = 2.5 s
      Propagation time = (3000 *103 m) / (3 × 108 m/s) = 0.01 s
      [ ⇒ speed of light = 3 * 108m/s]
      Latency = 0.000040 + 0.000060 + 2.5 + 0.01 = 2.5101 s
    4.   2.2 sec

  8. Suppose a channel having a bandwidth of 72 Kbps which uses pure ALOHA as a flow control mechanism. Assume there are N stations are trying to share this channel.     3.   776
      Explanation:

  11. Consider 2 hosts, H1, H2 are at two ends of a link. The transmission rate of the link is 2Mbps and the propagation delay is 10 msec. Host H1 will receive the data with a rate of 64Kbps from another link and send the data by dividing them into frames of length 56 bytes to Host H2. How much time elapses from the time a bit is received (from another link at Host H1) until the bit is stored (at Host H2)?
    Note : H2 stores the packet, once it receives the entire packet.( Marks: 0.00 )
    1.   15.5
    2.   18.2
    3.   16.4
    4.   17.2
      Explanation:
      Consider the first bit in a packet at Host H1. Before this bit can be transmitted, all of the bits in the packet must be received from another link. This requires
      56 * 8 / 64*103 = 7 msec.
      The time required to transmit the packet is
      56 * 8 / 2*106 = 224 sec.
      Propagation delay = 10 msec.
      The delay until the packet is stored at Host H2 is
      7 msec + 224 sec + 10 msec = 17.224msec


  12. Consider two ethernet channels having data rate of 190 kbps and 180 kbps respectively. If the channels used pulse stuffing TDM with no synchronization bits, What is the frame rate?
    Note: Pulse stuffing is often used as a means of controlling synchronization in systems that require both transmitter and receiver to transmit at the same bit rate.( Marks: 2.00 )
    1.   180K
    2.   170K
    3.   190K
      Explanation:
      We need to add extra bits to the second source to make both rates = 190 kbps. Now
      we have two sources, each of 190 Kbps.
      → The frame carries 1 bit from each source. Frame size = 1 + 1 = 2 bits.
      → Each frame carries 1 bit from each 190-kbps source. Frame rate = 190,000 frames/s.
    4.   185K


  13. Suppose the data 10011101 is transmitted in a ethernet link using the standard CRC method for error control. The Generator polynomial is x3 + 1. Suppose the 3rd bit from left is inverted while transmitting, what will be the CRC value at receiver end?( Marks: -0.66 )
    1.   010
    2.   100
      Explanation:
      The frame is 10011101. The generator is 1001.
      The message after appending three zeros is 10011101000.
      The remainder on dividing 10011101000 by 1001 is 100.
      So, the actual bit string transmitted is 10011101100.
      The received bit stream with an error in the third bit from the left is 10111101100.
      Dividing this by 1001 produces a remainder of 100.
    3.   010
    4.   001


  14. Consider a ethernet technology where all the stations can send their own size of data with a fixed size of header. Then what are the percentage of bits in a frame belong to the header for both the largest and smallest Ethernet frames.( Marks: 0.00 )

  15. Suppose there are 4 nodes N1, N2, N3 and N4 are using a slotted ALOHA channel to send their own huge data. When competing all the nodes to transfer their data only one node will get succeeded with a probability P. The time is divided into slots S1, S2, S3 ….and so on. What is probability of N1 succeeded in slot 5?( Marks: 0.00 )
    1.   4*(1-P)3(P(1-P)3)
    2.   P(1-P)3(1-P(1-P)3)
    3.   P(1-P)3(1-P(1-P)3)4
      Explanation:
      N1 succeeds in slot 5 for the first time means, N1 has to fail for 4 times and then 5th time it should succeed.
      I.e (1-PA)4PA
      Let PN1 is the probability of N1 success.[i.e N1 should success and remaining all (B, C, D) should fail
      → PN1 = P(1-P)(1-P)(1-P)
      ⇒ N1 succeeds for the first time in slot 5 = (1-PN1)4PN1
      = (1-(P(1-P)3))4P(1-P)3
    4.   4*P(1-P)3(1-P3(1-P)3)
