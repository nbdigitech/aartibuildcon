  2. Consider an array contains ‘n’ elements and we need to sort those elements using quick sort algorithm. Assume that we always choose median as the pivot element. What is the running time of this algorithm?( Marks: 1.00 )
    1.   O(n)
    2.   O(n2)
    3.   O(nlogn)
      Explanation:
      Algorithm is choosing n/2 smallest element as pivot. Hence the array is divided as.
      i) (n/2)–1 elements
      ii) median at n/2 nd location
      iii) n-(n/2) elements
      Therefore quick sort recurrence relation is given by
      T(n)=T((n/2)-1)+T(n-(n/2))+O(n)=O(nlogn)
    4.   None of these

  7. Consider we have m lists, each list consisting of n integers sorted in non-decreasing order. What is the time complexity to merge these lists into a single sorted list?( Marks: -0.66 )
    1.   O(mnlogn)
    2.   O(nmlogm)
      Explanation:
      Since m lists, each of size n. Each list is sorted in ascending order use directly merge procedure of merge sort algorithm.
      Take two list and merge, so one pair will take 2n time.
      So total pairs in first level will be m/2. So, total cost for one level is (m/2)*2n = mn
      In the next level, the cost for one pair is 4n and number of pairs will be m/4.. so next level cost will be mn.
      So, each level cost will be mn.
      So, we have logm levels then total time complexity is logm*(mn) = O(mnlogm)
    3.   O(n+m)
    4.   O(nm)

      left = 0

  13. Consider an array contains ‘n’ distinct elements and we need to sort them in non-decreasing order as follows: First find the minimum, remove this element from the array and find the minimum of the remaining elements, remove this element and so on until array become empty. In the best case, how many comparisons are needed?( Marks: -0.66 )
    1.   O(n)
    2.   O(n2)
    3.   O(nlogn)
      Explanation:
      Same as Heap sort i.e O(nlogn).

      Algorithm:
      Construct a min heap of given n elements.
      First find a minimum element take O(1) time, and delete min element.
      For finding the next minimum, you need to apply min-heapify algorithm on the remaining n-1 elements. So min-heapify algorithm takes O(log n) time.
      For n deletions it will take O(nlogn) time to sort all n elements.
    4.   None of these


  14. Consider an array contains 2k distinct elements. How many comparisons are needed to find the smallest and the second smallest elements in the given array?( Marks: 0.00 )
    1.   2k comparisons
    2.   2k-2 comparisons
    3.   2k + k -2 comparisons
      Explanation:
      A way to think about the solution is this is like a tennis match. Pair off the numbers and compare them
      Games: (a1,a2) (a3, a4), (a5, a6), (a7, a8)
      Winners: a12, a34, a56, a78
      Games: (a12, a34), (a56, a78)
      Winners: a1234, a5678
      Game: (a1234, a5678)
      Winner: a12345678
      Number of games = 7 ==> (2k - 1)
      The second best will have been defeated only by the winner. Suppose a3 is the winner. Then the second best will be a4, a12 or a5678.
      Games: (a4, a12)
      Winner: a412
      Games: a(412, 5678)
      Winner: a4125678
      So we have 2 games for the second-best ==> (log(2k) - 1)
      Hence number of games = 7 + 2 = 9 = (2k + log(2k) - 2) = 2k + k -2
    4.   k*2k comparisons


  15. Merge sort make two recursive calls. Which of the following statements is/are true after these two recursive calls finish, but before the merge step?( Marks: 2.00 )
    1.   The array elements form a heap
    2.   Elements in each half of the array are sorted amongst themselves
      Explanation:
      Merge sort is a divide-and-conquer algorithm.
      The recurrence relation of merge sort is T(n) = 2T(n/2) + O(n).
      Merge sort algorithm, divides the input array in to two halves. The merge() function is used for merging two halves. Time complexity to merge two sub arrays take O(n) time.
      Algorithm:
      1) If it is only one element in the list it is already sorted, return
      2) Divide the list recursively into two halves until it can no more be divided
      3) Merge the smaller lists into new list in sorted order.
    3.   Elements in the first half of the array are less than or equal to elements in second half of the array
    4.   All of the above
