  1. Identify the correct statement with respect to test set lock solution?( Marks: 0.00 )
    1.   Deadlock free
      Explanation:
      Deadlock free → If a process P1 is executing TSL instruction and entered into critical section and another process P2 with higher priority came then P1 is preempted and moved to ready queue, but lock is set. So, P2 is unable to enter the critical section, and P2 will move to ready queue and P1 comes to running state. Since P2 has higher priority again P1 is preempted and P2 starts executing. This cycle goes on and this condition is called livelock. But this condition is different from deadlock. In deadlock the processes involved will be in blocked state, but here the processes are in either ready state or running state but not doing any useful work.

      We cannot guarantee the processes to enter in FCFS order, because there is no fixed order to enter critical section. A process can enter critical section if it sees the lock is free or FALSE.

      →Starvation can occur → There is a chance that a process with lower priority may not get chance to acquire lock to enter into critical section.
    2.   Starvation free
    3.   Process enter critical section in FCFS order.
    4.   None

  11. Consider a schedule with 3 processes A,B and C, binary semaphores are S =1 , T=0 , Z=0 initially ;
    Process A
    {
    while(1)
    {
    P(S)
    printf("*");
    V(T)
    }
    }

    Process B
    {
    P(T)
    V(Z)
    }

    Process C
    {
    P(Z)
    V(S)
    }
    What is the maximum number of times * is printed?( Marks: 0.00 )
    1.   1
    2.   2
      Explanation:
      S is initialized to 1. So process A enters into critical section by setting S to 0, prints *, increments T to 1. Since there is a while loop in process A, it will wait for S to become 1. As soon as T becomes 1, process B enters its critical section by setting T to 0 and exits the critical section incrementing Z to 1. Now process C enters its critical section by setting Z to 0, and exits the critical section incrementing S to 1.
      Process A having while loop so it will print * one more time. Now there is no process to increment the value of S. So process A will keep on waiting for S to become 1.
      ∴ Answer is 2
    3.   >2
    4.   None
