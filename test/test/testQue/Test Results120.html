  1. Consider the following set F of functional dependencies on schema (A, B, C):

    A → BC

    B → C

    A → B

    AB → C

    The cannonical cover for F:( Marks: 0.00 )
    1.   {A → B, AB → C, B → C}
    2.   {A → B, A → C, B → C}
    3.   {A → B, B → C}
      Explanation:
      Given in relation R(A, B, C) FD set:

      {A → BC, B → C, A → B, AB → C}

      Step 1: Split FD’s such that RHS contain single attribute

      A → B

      A → C

      B → C

      A → B

      AB → C

      Step 2 : Find the redundant FD’s and delete them from the set.

      A → B

      B → C

      Step 3: Find the redundant attributes on LHS and delete them

      A → B

      B → C

      So, the minimal cover of the given FD set:

      {A → B, B → C}.

      Answer is option (c).
    4.   None of the above


  2. Given relation R(A, B, C, D) with FD set {AB → C, C → A, C → D, DE→A}.What is/are candidate key(s) of sub relation r1(ABCD) ?( Marks: 1.00 )
    1.   {AB, BD}
    2.   {AB, BC}
      Explanation:
      Given relation R(A, B, C, D, E) with FD set: {AB → C, C → A, C → D, DE→A}
      The FDs under sub relation r1 (ABCD) will be { AB → C, C → A, C → D}.

      Finding candidate key:
      B is not there in right hand side of any FD. So, B is necessary in every candidate key. We will check whether B itself is sufficient to be candidate key or not by finding (B)+. As (B)+ = B, B is not sufficient to be a candidate key. So, check for all combination of 2 attributes including B such as AB, BC, BD. Only AB and BC are candidate keys since (AB)+ = ABCD and (BC)+ = BCAD. Now Check for all combination of 3 attributes including B but excluding A and C( since AB and AC are already candidate keys any combination of attributes with them will become superkey). We cannot find any such combination. So, the candidate keys for the given relation are {AB, BC}. Answer is option (b).
    3.   {AB, BC, BD}
    4.   None of the above

  7. Consider a relation R(A, B, C, D, E). Which of the following FD set holds on R such that R is in 3NF but not in BCNF?( Marks: -0.66 )
    1.   {ABC → DE, B → C}
    2.   {A → BC, C → D, D → E}
    3.   {A → BCD}
    4.   None of the above
      Explanation:
      Checking for option (a):
      Given relation R(ABCDE) and FD set that holds on R: {ABC → DE, B → C}
      Finding Candidate key:
      Attributes A, B are not present in right hand side of the any FD. So, AB is necessary in every candidate key. We have to find whether AB is sufficient to be candidate key.
      So, find whether (AB)+ derives all attributes, since (AB)+ = ABCDE
      ∴ AB is necessary and sufficient to be a candidate key.
      No need to check for the combination of other attributes for finding candidate key.
      AB is the only candidate key of R
      Prime attributes: {A, B}
      Given FD’s
      ABC → DE ------ (1)
      B → C ------ (2)
      Check whether R satisfies 3NF:
      Formal definition of 3NF states that a relation R is said to be in 3NF if for every non-terminal FD X → Y in R, either X is a super key or Y is a prime attribute.
      Here FD (1) satisfies 3NF since LHS of FD(1) i.e., ABC is super key
      Here FD (2) does not satisfy 3NF since RHS of FD(2) i.e., C is NOT prime attribute
      ∴ Given relation R does not satisfy 3NF
      So, answer is option (a) is wrong.

      A → BC, ----- (1)
      C → D ----- (2)
      D → E ----- (3)
      Finding candidate key:
      Attribute A is not present in right hand side of any FD. So, A is necessary in every candidate key. So, find (A)+, (A)+ = ABCDE. A is sufficient to be a candidate key. Therefore we need not check for any other combination of attributes for candidate key. Only candidate key is A.
      Checking for 3NF:
      Here FD (1) satisfies 3NF since LHS of FD(1) i.e., A is super key in R
      FD(2) violates 3NF since either LHS of FD(2) i.e., C is not a super key or RHS of FD(2) i.e., D is not prime attribute.
      So, R does not satisfy 3NF
      So, option (b) is not answer

      Checking for option (c):
      Given relation R(ABCDE) and FD’s that hold on C,
      {A → BCD} ------ (1)
      Finding candidate key:
      Attribute E is not present in right hand side of any FD. So, E is necessary in every candidate key
      So, find (E)+ as (E)+ = E, E is not sufficient to be candidate key. Now check for 2 attribute combination including E as AE, BE, CE and DE, every (AE)+ determining all attributes. So, AE is the candidate key for R. similarly, check for all 3 attribute combinations and 4 attribute combinations. No other combination gives candidate key. Only AE is the candidate key for R.
      Checking for 3NF:
      FD(1) violates 3NF since LHS of FD(1) i.e., A is not super key and RHS of FD(1) i.e., BCD are not prime attributes. So, R does not satisfy 3NF.

  8. Given a relation R(MNPQT) and set of FD’s
    F1 = {M → TN, P → QM} and
    F2 = {M → N, P → Q, P → MT, MN → T}
    Which of the following statement is true?( Marks: 2.00 )
    1.   F1 and F2 are not equivalent
    2.   F1 and F2 are equivalent
      Explanation:
      Given relation R(MNPQT) and FD set
      F1 = {M → TN, P → QM} and
      F2 = {M → N, P → Q, P → MT, MN → T}
      F1 and F2 are equivalent if F1 can cover F2 and F2 can cover F1
      Check whether F2 can cover F1:
      In F1 first FD is M → TN
      ∴ find (M)+ in F2
      ⇒ (M)+ = MNT
      ∴ M → TN can be covered by F2
      Second FD is P → QM
      find (P)+ in F2
      (P)+ = PQMNT
      ∴ P → QM can be covered by F2
      ∴ We can say that F2 can cover F1 → (1)
      Check whether F1 can cover F2:
      In F2 first FD is M → N
      ∴ find (M)+ in F1
      ⇒ (M)+ = MNT
      ∴ M → N can be covered by F1
      Second FD is P → Q
      find (P)+ in F1
      (P)+ = PQMNT
      ∴ P → Q can be covered by F1
      Third FD is P → MT
      find (P)+ in F1
      (P)+ = PQMNT
      ∴ P → MT can be covered by F1
      Fourth FD is MN → T
      find (MN)+ in F1
      (MN)+ = MNT
      ∴ MN → T can be covered by F1
      ∴ We can say F1 can cover F2 → (2)
      From (1) & (2) we can say that F1 and F2 are equivalent.
      So, (a) is correct answer.
      ∴ We can say that F2 can cover F1 → (1)
    3.   We can’t say
    4.   None of the above


  9. Given a relational schema r(A, B, C, D, E) and set of FD’s {A → BC, CD → E, B → D, E → A} that holds on r. The decomposition of r into r1(A, B, C) and r2(A, D, E) is
    I.Lossless decomposition
    II.Dependency preserving
    Which of the following is true?( Marks: -0.66 )
    1.   I only
      Explanation:
      Given relation schema r(A, B, C, D, E) and FD set:
      {A → BC, CD → E, B → D, E → A}
      Given r is to be decomposed into r1(A, B, C) and r2(A, D, E).
      FDs under r1(A, B, C) will be {A → BC}.
      So, the key for r1 is {A}.
      FDs under r2(A D, E) will be {E → A}. So, the key for r2(A, D, E) is ED.
      ∴ Decomposition of r(A, B, C, D, E) into r1(A, B, C) and r2(A, D, E) is lossless since the common attribute of r1 and r2 i.e., A is key for r1.
      So, I is correct.
      Here the final FD set we obtained after decomposition is { A→BC, E→A}.
      The decomposition is not FD preserving since the FDs CD→E and B→D which are there in original FD set are not covered by final FD set ( (CD)+ = CDEAB in original FD set but in final FD set (CD)+ = CD and also (B)+ in original FD set is BD but (B)+ in final FD set is B). Therefore we can say that given decomposition is not FD preserving.
      So II is not correct.
      Answer is option (a).
    2.   II only
    3.   Both I & II
    4.   We can't say
