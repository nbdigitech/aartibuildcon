  1. Consider a hash table of size ‘n’ and keys are range from 0 to n2-1. What is the maximum number of distinct keys can be accommodated in the hash table, considering the following collision techniques: Chaining, Linear probing and Quadratic probing, respectively?( Marks: 0.00 )
    1.   n, n2, n
    2.   n2, n2, n2
    3.   n2, n, n
      Explanation:
      I. Chaining: With chaining, we can insert as many keys as there are in the universe which, in this case, is n2.
      II. Linear probing: Linear probing can only store as many elements as the size of the table which is n.
      III. Quadratic probing: Similar to linear probing, quadratic probing can only store as many elements as the size of the table which is n.
    4.   n, n, n


  2. Which of the following statements is/are TRUE?

    I.The collision resolution techniques by chaining and open addressing may result in the worst case search time for one of the ‘n’ keys in the hash table is O(n).

    II.A hash table always guarantees constant lookup time.( Marks: 1.00 )
    1.   Only I
      Explanation:
      I. True. All of the keys could hash to the same bucket (chaining), or all of the keys could have the same probe sequence (open addressing), resulting in a worst-case time of O(n) for a search.

      II. False: We cannot always guarantee a constant lookup time. If the collision resolution is done by chaining, then it may take linear time, O(n).
    2.   Only II
    3.   Both I and II
    4.   Neither I nor II


  3. Consider the Hash table contains ‘n’ keys and ‘k’ slots and the collision resolution is done by chaining. The chaining (set of keys which are hashing to the same slot), is implemented using AVL Tree instead of linked list. What is the worst time complexity of a key to insert in a hash table?( Marks: 0.00 )
    1.   O(n)
    2.   O(1)
    3.   O(log n)
      Explanation:
      Assume uniform hashing and use linked list, all the operations take O(1+ n) time, O(1) for applying hash function and access the slot, and O(n) is for inserting the linked list. Using AVL tree, we can reduce the term O(n) to O(log n). So, the overall time complexity is O(1+log n) = O(log n).
    4.   O(n log n)

  6. Using open addressing with linear probing, we sequentially insert three distinct keys k1, k2 and k3 into a hash table of size m. Assuming simple uniform hashing, what is the probability that we will need three probes, when inserting the third key, k3?( Marks: -0.66 )
    1.   3/m
    2.   2/m2
    3.   3/m2
      Explanation:
      k1 can go anywhere.

      k2 can either i) be placed immediately before or after k1 (or) ii) it will hit k1 on the first try and, thus by linear probing it will be placed immediately after k1.
      These are the only possible ways to get the first two keys in two adjacent cells in the hash table. This happens with probability 3/m. The only way for k3 to need three probes is when it hits the first of the two adjacent keys and this happens with probability 1/m. This gives us a total probability of 3/m2.
    4.   2/m

    4.   Separately apply binary search on the odd position elements and even position elements
      Explanation:
      Option (d) is correct:

      Algorithm:
      Calculate two mids m and m-1 respectively.
      If target element is greater than A[m-1] [ non-increasing array] then move the right of array to m-3 otherwise move the left of the array to m+1.
      If target element is greater than A[m] [ non-decreasing array] then move the left of array to m+2 otherwise move the right of the array to m-2.

  14. Linked Answer Questions Q14 & Q15
    Consider a Hash table containing ‘n’ keys and ‘k’ slots. Each key will hash into a slot in the given Hash table. (Assume collisions are resolved by chaining).
    What is the probability that the first slot of hash table will be empty?( Marks: 0.00 )
    1.   1-(1/k)
    2.   1/kn
    3.   1-(1/kn)
    4.   ((k-1)/k)n
      Explanation:
      Independently, each key has a probability of 1/k for hashing into the first slot (or) a probability of 1- 1/k for not hashing into the first slot. Thus, the probability that no key hashes into the first slot is (1-1/k)n=((k-1)/k)n.
