  2. How many comparisons are needed for a binary search in a set of 64 elements?( Marks: 0.00 )
    1.   12
    2.   13
    3.   14
      Explanation:
      Let T (n) be the number of comparisons needed in a binary search of a list of n elements.
      The recurrence relation is
      T(n) = T(n/2) + 2
      (2 comparisons because one to determine which half of the list to use and the other to find whether any elements of the list remain).
      Therefore T(1)= 2
      T(64) = T(32)+2 = T(16)+4 = T(8) + 6 = T(4) + 8 = T(2) + 10 = T(1) + 12 = 2+12 = 14
    4.   15


  3. Suppose that the function T satisfies the recurrence relation T(n) = 2T(√n) + 1 whenever n is a perfect square greater than 1 and f(2) = 1. Find a big-O estimate for T(n).( Marks: 0.00 )
    1.   O(√n)
    2.   O(logn)

      let n = 2k

      Assume T(2k) = S(m)

      a = 2, b= 2, k = 0

      = O(k)
      = O(log n)

  6. Consider an array containing ‘n’ elements. The elements present in an array are in arithmetic progression, but one element is missing in that order. What is the time complexity to find the position of the missing element using divide and conquer?( Marks: 2.00 )
    1.   O(n)
    2.   O(n2)
    3.   O(log n)
      Explanation:
      The time complexity is O (log n) using binary search.
      The idea is to go to the middle element. Check if the difference between middle and next to middle is equal to diff ((A[n-1] – A[0])/n) or not, if not then the missing element lies between mid and mid+1. If the middle element is equal to n/2th term in Arithmetic series (let n be the number of elements in input array), then missing element lies in right half. Else element lies in left half.
    4.   O(n log n)

  8. After applying few passes of quick sort on some give array, we got the following output: 1, 10, 5, 8, 25, 44, 55, 30, 70. How many pivot elements are there in this output?( Marks: 2.00 )
    1.   1
    2.   2
    3.   3
      Explanation:
      In the quick sort, every element to the left of pivot element should at most the pivot and every element to the right of pivot should be at least the pivot. If we check for this invariant in the given output array, it is easy to see that there are 3 elements that satisfy this invariant: 1, 25, and 70.
    4.   4

  10. Consider an array ‘A’ containing n distinct elements and those elements are in non-decreasing order. What is the time complexity to find the element in which A[i] = i using divide-and-conquer approach?( Marks: 2.00 )
    1.   O(n2)
    2.   O(n log n)
    3.   O(log n)
      Explanation:
      Apply binary search algorithm.

      First check whether middle element is (A[mid] = mid) or not. If it is, then return it; otherwise check whether index of middle element is greater than value at the index. If index is greater, then (A[i] = i) lies on the right side of the middle point . Else the A[i] =i element lies on left side.
    4.   O(n)

    Naveen has proposed a new data structure called “ARRAY BIN”. This data structure operations are similar to Array, you can INSERT values into it, and you can LOOKUP values to see if they are contained in the structure.

    An ARRAY BIN is implemented as a pair of lists, one is neat list and another one is messy list. The properties of those two lists are
    * The neat list is always in sorted order.
    * The messy list may or may not sorted order. The size of the messy list is √n.

    The LOOKUP algorithm for ARRAY BIN is as follows
    * Use a binary search to look a value in the neat list
    * For messy list, you need to iterate all values present in the list to find a value.

    The INSERT algorithm is as follows
    * A new value will insert into messy list
    * If the messy list Is too big then apply CLEAN-MESSY operation.

    The CLEAN-MESSY algorithm is as follows
    * Sort the messy list
    * Merge the sorted messy list with neat list
    * The messy list is empty now.
    What is the worst case asymptotic runtime of LOOKUP on a “ARRAY BIN”?( Marks: -0.66 )
    1.   Θ(log n)
    2.   Θ(n)
    3.   Θ(√n)
      Explanation:
      Binary search on the neat list takes Θ(log n), and iterating over the messy list takes Θ(√ n), for a total time of Θ(√ n).
    4.   Θ(n log n)

    * The neat list is always in sorted order.
    * The messy list may or may not sorted order. The size of the messy list is √n.

    The LOOKUP algorithm for ARRAY BIN is as follows
