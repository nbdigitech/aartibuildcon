  2. Consider the following two statements:
    P: A regular grammar cannot be ambiguous
    Q: Every regular set has a LR(1) grammar.
    Which of the following is TRUE?( Marks: -0.33 )
    1.   P is true and Q is false
    2.   Both P and Q are false
    3.   P is false and Q is true
      Explanation:
      Checking for statement P: A regular grammar cannot be ambiguous
      A regular grammar can also be ambiguous.
      For eg: consider the following grammar:
      S → rR--------- (production I)
      / r --------- (production II)
      R → rR--------- (production III)
      / ϵ -------- (production IV)
      In the above grammar there are two leftmost left most derivations for string ‘r’
      (1) S → rR ------- (using production I, S → rR)
      S → r ------- (using production IV, R → ϵ)
      (2) S → r--------- (production II, S → r)
      So, statement P is false.
      Checking for Q: Every regular set has a LR(1) grammar
      LR(1) grammars are those grammars which can be parsed by LR parsers having one look ahead.
      LR parsers handle DCFL in linear time.
      All regular languages are also DCFL hence, they all can be parsed by a LR(1) grammar.
      So, statement Q is True.
    4.   Both P and Q are true

  5. The equivalent production rules corresponding to the following production rules:
    E → E + T | T
    T → T * F |F
    F→ (E) | id( Marks: 1.00 )
    1.   E → TE', E' → +TE' |∈, T → FT', T' → *FT |∈, F → (E) | id
    2.   E → T + E| T, T → F*T | F, F → (E) | id
    3.   E → T, E' →E + T, T→ F, T → T * F, F → (E) | id
    4.   E → TE', E' → +TE' | ∈, T → FT', T' → *FT' | ∈, F → (E) | id
      Explanation:
      The given grammar is left recursive.
      A grammar is said to be left recursive if it is of the form:
      A → A α | β
      The grammar after left recursion is:
      A → β A'
      A' → ∈ | α A'
      The given grammar:
      E → E + T ------- (1)
      | T ------- (2)
      T → T * F ------- (3)
      | F ------- (4)
      F→ (E) ------- (5)
      | id ------- (6)
      has two left recursions at production (1) and production (3)
      After removing the left recursion we get:
      E → TE'
      E' → +TE' |∈
      T → FT'
      T' → *FT' |∈
      F → (E) | id

  9. For an object oriented programming consider the following example:
    (1) There is a class C with a method named m( )
    (2) D is a sub class of C, and D has its own method named m( )
    (3) There is a use of m of the form x.m( ), where x is an object of class C.
    For statement (3) whether x will be a class of C or of the subclass of D when will it be resolved?( Marks: -0.66 )
    1.   Run time
      Explanation:
      Normally, it is impossible to tell at compile time whether x will be of class C or of subclass D. If the method application occurs several times it is likely that some will be on objects denoted by x that are in class C but not in D, while others will be in class D.
      It is not until run-time that it can be decided which definition of m is the right one. It is after compilation, when the code is generated, during run time, it is determined to which class the object x belongs and the call to one or other method named m is resolved.
    2.   None of these
    3.   Compile time
    4.   Edit time
