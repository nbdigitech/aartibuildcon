    An ARRAY BIN is implemented as a pair of lists, one is neat list and another one is messy list. The properties of those two lists are
    * The neat list is always in sorted order.
    * The messy list may or may not sorted order. The size of the messy list is √n.

    The LOOKUP algorithm for ARRAY BIN is as follows
    * Use a binary search to look a value in the neat list
    * For messy list, you need to iterate all values present in the list to find a value.

    The INSERT algorithm is as follows
    * A new value will insert into messy list
    * If the messy list Is too big then apply CLEAN-MESSY operation.

    The CLEAN-MESSY algorithm is as follows
    * Sort the messy list
    * Merge the sorted messy list with neat list
    * The messy list is empty now.
    What is the worst case asymptotic runtime of LOOKUP on a “ARRAY BIN”?( Marks: -0.66 )
    1.   Θ(log n)
    2.   Θ(n)
    3.   Θ(√n)
    4.   Θ(n log n)
